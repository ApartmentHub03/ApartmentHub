## Why this is blocked

Building backlinks is an **off-page / external** SEO activity. A backlink is created when another website links to ApartmentHub.nl. No change to this codebase can produce a backlink — backlinks are produced by:

- Outreach to authoritative Dutch real estate sites (Funda, Pararius, Huurwoningen.nl, NVM, etc.)
- Directory submissions (real-estate associations, expat services, city portals)
- PR / press coverage in real-estate or expat publications
- Guest posts and editorial placements
- Partnerships with property managers, relocation agents, universities, embassies
- Sponsorships and event listings

All of these require humans doing manual relationship-building, content marketing, or paid PR work outside the repository.

## What is *not* a fix

It would be tempting to implement something tangentially related (adding outbound links to authoritative sites, adding `rel="me"` markup, adding a "Partners" page with no real partners, etc.), but none of these would actually generate backlinks — they would just be code churn that misrepresents the dashboard issue as resolved. Per the dispatch rules ("STAY IN SCOPE" / "do NOT guess"), I am not implementing a substitute task.

## Recommended human follow-up

A marketing/SEO owner should:

1. Identify a target list of 20–50 authoritative .nl real-estate and expat-relocation domains.
2. Run an outreach campaign (cold email, partnership offers, guest-post pitches).
3. Submit ApartmentHub to relevant directories (NVM partner directory, IamExpat, Expatica resource lists, Amsterdam city housing pages).
4. Track acquired backlinks in a tool like Ahrefs / Semrush and feed that data back to the SEO dashboard.

Once concrete *on-page* deliverables are scoped (e.g. "create a /partners page listing X confirmed partners", "add JSON-LD `sameAs` references to confirmed partner profiles"), those can be dispatched as separate, code-actionable jobs.
